# 多模态模型攻击与安全评估平台：演示操作流程和运行流程

## 1. 本地运行流程

### 1.1 安装依赖

```bash
pip install -r requirements.txt
```

### 1.2 启动前端平台

```bash
python start_frontend.py
```

或直接运行：

```bash
streamlit run src/services/frontend/app.py
```

浏览器打开 Streamlit 显示的本地地址，通常是：

```text
http://localhost:8501
```

### 1.3 数据目录

平台首次启动会自动创建：

```text
data/
data/uploads/          # 用户上传的原始样本
data/adversarial/      # 自动生成的对抗样本
data/exports/          # 导出文件
data/platform.sqlite3  # SQLite 数据库
```

数据库保存项目、样本、实验、结果、指标和状态；文件夹只保存上传文件与生成文件。

## 2. Render 部署流程

仓库已包含 `render.yaml` 和 `.python-version`。

Render 推荐配置：

```text
Build Command:
pip install -r requirements.txt

Start Command:
streamlit run src/services/frontend/app.py --server.address 0.0.0.0 --server.port $PORT --server.headless true

Python Version:
3.11.9
```

如果 Render 没有自动读取 `.python-version`，在 Environment 中手动添加：

```text
PYTHON_VERSION=3.11.9
```

## 3. 演示操作流程

### 3.1 新建实验项目

1. 打开左侧功能模块，进入“样本管理”。
2. 在“新建项目”中输入项目名称，例如：

```text
VQA 黑盒迁移测试
```

3. 输入项目说明。
4. 点击“创建项目”。
5. 刷新页面后，在左侧“当前实验项目”中选择新项目。

### 3.2 上传用户样本

1. 进入“样本管理”。
2. 在“上传样本”区域选择文件。
3. 支持上传：
   - 图像：`png`、`jpg`、`jpeg`、`webp`
   - 文本：`txt`、`md`、`json`、`csv`
   - 音频：`wav`、`mp3`、`m4a`、`aac`
   - 视频：`mp4`、`mov`、`avi`、`mkv`、`webm`
4. 可填写样本标签、真实类别、问答答案或目标答案。
5. 点击“保存上传样本”。
6. 下方“样本列表”会展示：
   - 样本 ID
   - 文件名
   - 模态
   - 标签
   - 存储路径
   - 上传时间
7. 图像样本会在“预览”区域显示缩略图。

### 3.3 运行白盒攻击

1. 进入“攻击实验”。
2. 填写实验名称。
3. 选择任务场景，例如“图像分类”。
4. 攻击模式选择“白盒攻击”。
5. 攻击算法选择 `FGSM`、`PGD`、`CW` 或 `Ensemble`。
6. 替代模型 / 白盒模型选择：
   - ResNet
   - ViT
   - Ensemble
   - CLIP
   - BLIP
7. 选择一个或多个上传样本。
8. 设置扰动强度：
   - `eps4`
   - `eps8`
   - `eps16`
   - 自定义 epsilon
9. 设置迭代次数、样本数和目标标签。
10. 点击“运行攻击实验”。
11. 平台会自动生成对抗样本，并保存结果。

### 3.4 运行黑盒迁移攻击

1. 进入“攻击实验”。
2. 攻击模式选择“黑盒迁移”。
3. 选择替代模型，例如 `ResNet` 或 `ViT`。
4. 选择目标模型，例如：
   - DenseNet
   - MobileNet
   - EfficientNet
   - CLIP
   - BLIP
   - LLaVA
   - Qwen-VL
5. 设置攻击算法、扰动强度、迭代次数和样本数。
6. 点击“运行攻击实验”。
7. 平台会记录：
   - 原始输出
   - 攻击后输出
   - 是否输出偏移
   - 是否攻击成功
   - 黑盒查询次数
   - 对抗样本路径
   - 运行日志

### 3.5 查看指标图表

进入“工作台”查看：

- 当前项目样本数
- 实验数量
- 平均 ASR
- 黑盒查询次数
- 最近实验列表
- 实验 ASR 柱状图

### 3.6 检索、归档和复现实验

1. 进入“项目与复现”。
2. 可查看全部项目。
3. 可归档项目。
4. 可按实验名称检索实验。
5. 可归档实验。
6. 可点击“复制为复现实验”，复制历史实验配置。
7. 可查看每个实验的结果明细和日志。

### 3.7 导出结果

进入“导出与日志”。

可下载：

- 完整 JSON 报告
- 攻击结果 CSV

日志表包含：

- 运行时间
- 实验 ID
- 样本 ID
- 攻击日志

## 4. 当前功能覆盖情况

### 已完成

- 用户样本上传闭环
- 样本 ID 自动生成
- 样本保存到 `data/uploads/`
- 样本列表、标签、路径、上传时间展示
- 图像样本预览
- 多样本选择进入实验
- 白盒 / 黑盒迁移攻击模式选择
- FGSM / PGD / CW / Ensemble 攻击配置
- 替代模型和目标模型选择
- eps4 / eps8 / eps16 / 自定义 epsilon
- 迭代次数、样本数、目标标签配置
- 自动生成对抗样本并保存到 `data/adversarial/`
- 实验项目管理
- 实验检索、归档、复现配置复制
- SQLite 持久化保存项目、样本、实验、结果和指标
- 图文检索、VQA、Caption、音频、视频任务流程入口
- CLIP、BLIP、LLaVA、Qwen-VL 接口配置入口

### 当前实现边界

当前版本优先保证平台闭环、数据结构和演示流程完整。

图像扰动攻击已生成真实对抗图片文件，但模型输出目前使用确定性伪预测器记录流程和指标，便于无 GPU、无外部 API 的环境稳定演示。

图文检索、VQA、图像描述、音视频攻击已经具备样本管理、实验配置、记录和导出流程；接入真实模型时，只需要替换预测函数，并保留现有 SQLite 表结构和前端流程。

## 5. 接入真实多模态模型的扩展点

主要替换位置：

```text
src/services/frontend/app.py
```

建议替换函数：

```python
pseudo_prediction(...)
```

可扩展为：

- CLIP 图文检索相似度
- BLIP 图像描述生成
- LLaVA / Qwen-VL 视觉问答
- 文本诱导攻击后的回答偏移检测
- 音视频模型输出偏移检测

建议真实模型服务返回统一结构：

```json
{
  "model": "Qwen-VL",
  "task": "视觉问答 VQA",
  "output": "模型输出",
  "score": 0.87,
  "tokens": 128,
  "latency_ms": 950
}
```

然后写入 `results.metrics_json`，即可复用现有报表、日志和导出功能。
